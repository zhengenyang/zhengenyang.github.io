# Agents Management

## GLM-5 Tool Use 核心改进总结

> 数据来源：智源社区 | 更新时间：2026-02

### 一、工具调用能力突破

| 指标 | GLM-5 表现 |
|------|-----------|
| 工具调用次数 | 700次精准调用 |
| 调用准确率 | 98.2% |
| 上下文切换 | 800次无损切换 |
| 持续运行时长 | 24小时不间断 |

### 二、核心技术改进

#### 1. 自主工具调用能力
- **无需人工引导**：自主判断调用时机、选择工具类型、校验执行结果
- **全场景覆盖**：代码编译、数据处理、文件操作、逻辑校验
- **调用准确率 98.2%**：第700次与首次调用的语法规范、执行准确率毫无差异

#### 2. 长程任务执行能力
- **智能上下文缓存**：切换不丢帧、重启不跑偏
- **状态持久化机制**：历经数百次上下文重置，仍能锁定最终目标
- **解决行业三大顽疾**：长任务崩断、上下文遗忘、执行中断

#### 3. 评测基准表现（开源第一）

| 评测基准 | 维度 | GLM-5 排名 |
|---------|------|-----------|
| MCP-Atlas | 大规模端到端工具调用 | 开源第一 |
| τ²-Bench | 复杂多工具场景规划执行 | 开源第一 |
| BrowseComp | 联网检索与信息理解 | 开源第一 |
| Vending Bench 2 | 长期经营规划 | 开源第一（$4,432）|

#### 4. 技术架构支撑
- **"Slime"异步强化学习框架**：支持更大规模模型与复杂RL任务
- **异步智能体强化学习算法**：从长程交互中持续学习
- **DeepSeek Sparse Attention**：降低部署成本，提升Token效率
- **参数规模扩展**：744B总参数，40B激活参数
